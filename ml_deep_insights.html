Chapter 01 · Foundations

What ML really is — the inversion of programming

Machine learning is not magic. It is a systematic method for finding patterns in data — by optimising numbers, not by writing rules. Understanding this one idea precisely makes everything else coherent.

In traditional software, a human writes explicit rules: "if the email contains 'free money', mark it spam." This works for simple problems. But the world is full of problems where the rules are too complex, too numerous, or simply unknown. You cannot write rules that recognise handwriting. You cannot write rules that translate languages. The search space of possible rules is infinite.
Machine learning solves this by flipping the process. Instead of rules → output, you provide examples of inputs + correct outputs, and the algorithm discovers the rules itself. The "rules" it discovers are encoded not as readable code, but as millions of numerical parameters — called weights — inside a mathematical function.

Traditional programming

Input: Rules + Data
Output: Answers

You define logic. Computer executes it. Breaks the moment reality doesn't fit your rules.

Machine learning

Input: Data + Answers
Output: Rules (learned)

You curate examples. Algorithm discovers logic. Generalises to new cases automatically.
The deep insight: When you train a model to classify dog photos, it doesn't learn a definition of "dog." It learns to detect edges, then textures, then shapes, then configurations of shapes that correlate with the label "dog." The resulting function is not interpretable — it is emergent. This is both ML's greatest power and its greatest limitation.

When to use ML and when not to

ML is not always the right tool. It is the right tool when: (1) the problem has patterns that are too complex to code manually, (2) you have labelled examples to learn from, and (3) the pattern is consistent enough to generalise. It is the wrong tool when simple rules work, when you have almost no data, or when you need a fully explainable decision process for legal or ethical reasons.

The vocabulary that unlocks everything

Eight terms explain 95% of ML conversations. Master these and research papers, job descriptions, and tool docs become readable.

You feed features from a dataset through a model, it makes a prediction, the loss function measures how wrong it was, and training adjusts the weights to reduce that loss — until the model is good enough to use for inference on new data.

Batches, epochs, and iterations

Three terms describe the scale of training. A batch is a small subset of the dataset processed in one forward pass (e.g. 32 or 64 examples). An iteration is one pass through one batch. An epoch is one complete pass through the entire training dataset. A model might train for 100 epochs, each containing thousands of iterations.
Batch size
How many examples per step. Smaller = noisier but faster; larger = smoother but slower. Typically 32–256.
Epoch
One full pass through all training data. Underfitting = too few epochs. Overfitting = too many.
Learning rate
How big each weight update step is. The single most important hyperparameter. Too high = explodes. Too low = never converges.
# Simplified training loop in PyTorch
for epoch in range(100):
  for batch in dataloader:
    X, y = batch                     # features and labels
    predictions = model(X)         # forward pass
    loss = loss_fn(predictions, y) # compute loss
    loss.backward()                # backpropagation
    optimizer.step()               # update weights
    optimizer.zero_grad()          # reset gradients
Chapter 04 · Core mechanics

Neural networks — anatomy and intuition

A neural network is a layered mathematical function loosely inspired by the brain. Each layer transforms the data, and together they learn arbitrarily complex patterns.

Every neural network has three types of layers. The input layer receives the raw features. Hidden layers transform the data through a series of weighted sums and non-linear activations. The output layer produces the final prediction — a number, a class, a token.
Anatomy of a single neuron
xᵢ = inputs (features)
wᵢ = weights (learned)
b = bias (learned)
activation = non-linearity

Activation functions — the non-linearity that makes depth useful

Without activation functions, stacking multiple layers is mathematically equivalent to a single layer. Activations introduce non-linearity, which allows networks to approximate any function. Three dominate modern practice:
ActivationFormulaBehaviourUsed where
ReLUmax(0, x)Zero for negatives, linear for positives. Fast, simple.Hidden layers in most modern networks
Sigmoid1 / (1+e⁻ˣ)Squashes output to (0,1). Interpretable as probability.Binary classification output layers
Softmaxeˣⁱ / ΣeˣʲConverts logits to probabilities that sum to 1.Multi-class classification outputs

Key insight: "Deep" learning means using many layers (depth). Each layer learns progressively more abstract representations. In vision: layer 1 detects edges, layer 5 detects shapes, layer 20 detects concepts like "dog snout." This hierarchical feature learning is why depth is so powerful.
Chapter 05 · Core mechanics

Gradient descent — the engine of learning

Gradient descent is the algorithm that makes training possible. It answers the question: given that the model is wrong, how should each weight change to make it less wrong?

Imagine the loss function as a landscape of hills and valleys. The model's current weights are a ball sitting somewhere on that landscape. The goal is to roll the ball to the lowest point — the minimum loss. Gradient descent does this by asking: which direction is downhill? Then it takes a small step in that direction.
Mathematically, the gradient is the derivative of the loss with respect to each weight. It tells you the slope at your current position. A positive gradient for a weight means "increasing this weight increases loss, so decrease it." A negative gradient means "decrease this weight to increase loss, so increase it." You step in the opposite direction of the gradient.

Vanilla gradient descent uses the same learning rate for every weight. Adam (Adaptive Moment Estimation) gives each weight its own adaptive learning rate based on recent gradient history. It is the default optimiser for most deep learning tasks and is why you rarely need to tune the learning rate much in practice.
Chapter 06 · Core mechanics

Backpropagation — how error flows backwards

Backpropagation is the algorithm that computes how much each weight contributed to the loss — so gradient descent knows exactly how to adjust each one. Without it, training a deep network would be computationally infeasible.

The forward pass computes a prediction. The backward pass asks: if I change weight W by a tiny amount, how much does the loss change? This is computed using the chain rule of calculus, applied backwards through every layer of the network. The result is a gradient for every single weight — which is then used by the optimiser to update them.
The chain rule in plain English

If the final loss depends on layer 3, and layer 3 depends on layer 2, and layer 2 depends on a weight W, then the effect of W on loss = (effect of W on layer 2) × (effect of layer 2 on layer 3) × (effect of layer 3 on loss). Backprop automates this multiplication across all layers simultaneously. PyTorch and TensorFlow compute this for you with loss.backward().

Why this matters for you as an engineer: You almost never implement backprop manually — deep learning frameworks do it automatically. But understanding what it does (distribute blame for errors back to each weight) helps you debug training failures. If gradients become zero (vanishing gradient) or astronomically large (exploding gradient), training breaks. Understanding backprop is how you diagnose these.

Vanishing and exploding gradients

In deep networks, gradients are multiplied as they travel backwards through layers. If each multiplication makes them slightly smaller (common with sigmoid activations), they become nearly zero by the time they reach early layers — those layers stop learning. This is the vanishing gradient problem. The exploding gradient problem is the opposite — gradients grow exponentially and destabilise training. Modern solutions include: ReLU activations (reduces vanishing), residual connections (skip connections in ResNets), layer normalisation, and gradient clipping.
Chapter 07 · Core mechanics

Overfitting, underfitting and the bias-variance tradeoff

The central challenge of ML is not making a model that performs well on training data — that is easy. The challenge is making a model that generalises to new, unseen data. This is where most real-world ML projects fail.

Underfitting
Model too simple. Misses even the obvious pattern. High error on both training and test data.
Good fit
Model captures the underlying pattern. Low training error. Generalises well to new data.
Overfitting
Model memorised training data including noise. Performs terribly on any new data.

The bias-variance tradeoff

Bias is error from wrong assumptions — the model is too simple to capture the true pattern (underfitting). Variance is error from sensitivity to small fluctuations in training data — the model is too complex and memorises noise (overfitting). You cannot eliminate both simultaneously by simply changing model complexity. The art of ML is navigating this tradeoff.

Regularisation — the solutions to overfitting

TechniqueWhat it doesWhen to use
L2 regularisationPenalises large weights by adding their squared value to loss. Keeps weights small.Default regularisation for most models
DropoutRandomly zeroes out neurons during training. Forces the network not to rely on any single path.Fully connected layers in neural networks
Early stoppingStop training when validation loss stops improving, before the model overfits.Any training loop with a validation set
Data augmentationArtificially expand training data by transforming existing examples (rotate, crop, flip images).Computer vision — whenever data is limited
Batch normalisationNormalises layer inputs during training. Stabilises learning and acts as mild regulariser.Deep networks, often between layers
Chapter 08 · Algorithms

Core ML algorithms explained

Before deep learning dominated, classical ML algorithms solved (and still solve) a huge range of problems. Every ML engineer should know these — they are faster, more interpretable, and often better than neural networks for structured/tabular data.

01
Linear Regression
Supervised · Regression
Finds the best-fit line (or hyperplane) through data points. Predicts a continuous value. The simplest model — and often a surprisingly strong baseline. Core idea: minimise the sum of squared differences between predictions and actual values. Every weight is directly interpretable as "how much does this feature move the output?"
→ Use for: house prices, demand forecasting, any continuous output with linear relationships
02
Logistic Regression
Supervised · Classification
Despite the name, it is a classification algorithm. Outputs a probability (0 to 1) using the sigmoid function. Decision boundary: predict class 1 if probability > 0.5. Core idea: model the log-odds of the outcome as a linear function of features. Fast, interpretable, and often competitive with complex models.
→ Use for: spam detection, medical diagnosis, credit scoring, any binary yes/no outcome
03
Decision Trees & Random Forests
Supervised · Both
A decision tree asks a series of yes/no questions about features to reach a prediction — like a flowchart. Single trees overfit easily. A random forest trains hundreds of trees on random subsets of data and features, then averages their predictions. This ensemble dramatically reduces variance. XGBoost (gradient boosted trees) is the state-of-the-art for tabular data and wins most Kaggle competitions on structured data.
→ Use for: tabular/structured data, business intelligence, any dataset with mixed feature types
04
Support Vector Machines (SVM)
Supervised · Classification
Finds the hyperplane that maximally separates two classes — maximising the margin between them. The "support vectors" are the data points closest to the boundary. With the kernel trick, SVMs can learn non-linear boundaries by implicitly mapping data to higher dimensions. Powerful but does not scale well to very large datasets.
→ Use for: image classification (pre-deep learning), text classification, small-to-medium datasets
05
K-Means Clustering
Unsupervised · Clustering
Groups data into K clusters by iteratively assigning each point to the nearest centroid, then recomputing centroids. No labels needed. Core idea: minimise the within-cluster sum of squared distances. Simple, scalable, but requires you to specify K in advance and assumes spherical clusters.
→ Use for: customer segmentation, document grouping, anomaly detection, image compression
06
k-Nearest Neighbours (kNN)
Supervised · Both
The simplest possible learning algorithm: to classify a new point, find its k nearest training examples and take a vote. No training phase — all computation happens at inference. Core idea: similar inputs have similar outputs. Interpretable but slow for large datasets and breaks down in high-dimensional spaces (curse of dimensionality).
→ Use for: recommendation systems, anomaly detection, baseline comparisons
Chapter 09 · Algorithms

The three types of ML — supervised, unsupervised, RL

These three paradigms differ fundamentally in what information is available during training and what the algorithm is optimising for.

Supervised learning

You have labelled examples: (input, correct_output) pairs. The model learns a mapping from input to output by minimising prediction error. This is the dominant paradigm — most commercial AI is supervised. Requires labelled data, which is often the bottleneck.
Examples: image classification, spam detection, price prediction, translation, LLMs (trained on next-token prediction — a self-supervised variant)

Unsupervised learning

No labels. The model finds hidden structure — clusters, patterns, compressions, or representations — in unlabelled data. Often used as a preprocessing step (learning good representations) before supervised fine-tuning. Word embeddings (Word2Vec, GloVe) are a classic example.
Examples: customer segmentation, topic modelling, anomaly detection, dimensionality reduction (PCA, t-SNE), generative models

Reinforcement learning (RL)

An agent takes actions in an environment and receives rewards or penalties. The goal is to learn a policy — a mapping from state to action — that maximises cumulative reward over time. No labelled examples: the signal is entirely from the reward function. Notoriously difficult to train but capable of superhuman performance.
Examples: AlphaGo/AlphaZero, robot locomotion, game playing, RLHF in LLMs (making models follow instructions and be safe)
RLHF — how ChatGPT and Claude become "helpful": LLMs are first pretrained with self-supervised learning (predict the next word). Then humans rate outputs, training a reward model. Then RL fine-tunes the LLM to maximise the reward model's score. This Reinforcement Learning from Human Feedback (RLHF) is the step that turns a raw language model into an assistant that actually follows instructions.
Chapter 10 · Algorithms

Deep learning — why depth matters

Deep learning is ML with neural networks containing many layers. The depth allows the model to learn hierarchical representations automatically — and this changes everything.

The key insight of deep learning is the universal approximation theorem: a neural network with sufficient width and depth can approximate any continuous function to arbitrary precision. In practice, depth (many layers) is far more efficient than width (one very wide layer) for learning complex functions.

The major deep learning architectures

ArchitectureKey innovationBest for
Convolutional Neural Networks (CNN)Convolutional filters share weights across spatial positions — detect features anywhere in an image.Images, video, any spatial data
Recurrent Neural Networks (RNN/LSTM)Hidden state carries information across time steps. Memory of sequences.Sequential data, time series (now mostly replaced by Transformers)
TransformersSelf-attention: every token attends to every other token simultaneously. Scales massively.Language, code, images (ViT), audio — the dominant architecture
Generative Adversarial Networks (GAN)Two networks compete: generator creates fakes, discriminator detects them. Generator improves until indistinguishable.Image synthesis, data augmentation
Diffusion ModelsLearn to reverse a noise-addition process. Generates samples by iteratively denoising from random noise.Image/audio generation — Stable Diffusion, DALL-E 3, Sora
Transfer learning — the most important practical insight in deep learning: You almost never train from scratch. A model pretrained on millions of images or billions of words already understands the world at a deep level. You fine-tune it on your specific task with far less data and compute. This is how 99% of real-world deep learning projects work. It's why Hugging Face exists.

Transformers & LLMs — how ChatGPT and Claude work

The Transformer architecture (introduced in the 2017 paper "Attention Is All You Need") is the foundation of every major AI system today. Understanding it is essential for any AI engineer.

Self-attention — the core idea

In a sentence like "The animal didn't cross the street because it was too tired" — what does "it" refer to? Humans know it's the animal, not the street. Self-attention gives the model this ability: every word can look at every other word and decide how much to "attend" to it when building its own representation.

How LLMs are trained — the full pipeline

Stage 1
Pretraining — self-supervised on massive text
The model is trained to predict the next token across trillions of words from the internet, books, and code. No human labels needed — the data labels itself. This gives the model broad world knowledge and language structure. Requires enormous compute (weeks on thousands of GPUs).
Stage 2
Supervised fine-tuning (SFT)
Human contractors write thousands of high-quality example conversations — "question: ... answer: ...". The model is fine-tuned on these to learn the format and style of being a helpful assistant. This is where "talking to an AI" behaviour comes from.
Stage 3
RLHF — alignment via human preferences
Humans rank pairs of model outputs (which is better?). A reward model is trained on these preferences. Then RL (PPO algorithm) fine-tunes the LLM to maximise the reward model's score. This is how Claude learns to be helpful, honest, and harmless — not just fluent.
What LLMs don't do: They are not databases. They do not "look up" facts — they predict the statistically likely continuation of text based on training patterns. This is why they hallucinate. Understanding this is critical for building reliable AI applications — which is why RAG (Retrieval Augmented Generation) exists: you give the model the facts, and it reasons over them.

Evaluation metrics — how you know if a model is good

Accuracy alone is almost always the wrong metric. Understanding which metric matches your business problem is a core engineering skill.

MetricFormula (simplified)When to useWatch out for
AccuracyCorrect / TotalBalanced classes, equal cost of errorsUseless on imbalanced datasets (95% accuracy detecting rare cancer = 0 useful)
PrecisionTrue Pos / (True Pos + False Pos)Cost of false positives is high (spam filter — don't delete real email)Can be gamed by being very conservative
RecallTrue Pos / (True Pos + False Neg)Cost of false negatives is high (cancer screening — missing a case is catastrophic)Can be gamed by predicting everything positive
F1 ScoreHarmonic mean of precision and recallImbalanced classes where both precision and recall matterWeights precision and recall equally — adjust with F-beta if they're not
ROC-AUCArea under ROC curveRanking quality of binary classifier across all thresholdsCan be misleading on heavily imbalanced datasets
RMSE√(mean squared error)Regression problems — penalises large errors heavilySensitive to outliers
MAEMean absolute errorRegression — when outliers shouldn't dominateLess differentiable than RMSE near zero
Perplexityexp(cross-entropy loss)Language models — how "surprised" the model is by dataOnly comparable across models with the same tokenisation
Train / validation / test splits: Always split your data into three sets. Train on the training set. Tune hyperparameters on the validation set. Evaluate the final model exactly once on the test set. Evaluating on the test set during development leaks information and gives an optimistic — false — estimate of real-world performance.
